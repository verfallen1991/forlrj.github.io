git

基本概念

工作目录

索引区(index)

HEAD：上一次提交的快照

本地仓库

基本命令

git init

  • git init：初始化目录为git目录

git add

  • git add file：将file添加到暂存区
  • git add -A

git commit

  • git commit：将暂存区文件提交到本地资源区
  • git commit –amend：添加说明 or 改变说明

git branch

  • git branch：显示所有分支
  • git branch bugfix：创建bugfix分支
  • git branch -d bugfix：删除bugfix分支

git checkout：可以切换

  • git checkout bugfix：切换到bugfix分支
  • git checkout commitId：切换到commitId即分离HEAD
  • git checkout -b bugfix：创建并切换到bugFix分支
  • git checkout – file1：暂存区文件覆盖工作区
  • git checkout file1：

git merge

  • git merge bugfix：将bugfix分支合入当前分支

    git rebase

    合并几个commit

  • git rebase -i BeginCommitId EndCommitId：将开始到结束(左开右闭)的commit合并，EndCommitId不写默认是到HEAD

    将几个commit加入其它branch

  • git rebase BeginCommitId EndCommitId –onto master：将开始到结束的commit加入到master分支
    注意：需要git reset –hard NewCommitId，才能将HEAD Index WorkDir都更新到此Commit

git reset

– git reset –mixed(默认) file1：本地仓库文件覆盖HEAD、暂存区
– git reset –hard commitid：commitid的提交覆盖HEAD、暂存区和工作区
– git reset –soft commitid：commitid的提交覆盖HEAD

git stash

  • git stash
  • git stash save “fugfix”
  • git stash pop
  • git stash pop stash@{0}
  • git stash apply：不删除stash节点
  • git stash list
  • git stash drop stash@{0}

git pull

git fetch

git push

  • git push
  • git push origin bugfix

git revert

  • git revert HEAD~：撤销上一次commit，前后都保留

git log

  • git log -p
  • git log –stat

git diff

  • git diff：工作目录与暂存区的差别
  • git diff –staged：暂存区与上一次commit的差别
